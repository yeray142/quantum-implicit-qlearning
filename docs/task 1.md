## 1. Context and Motivation

### 1.1 The Offline RL Problem

Offline RL (also called batch RL or full batch RL) learns a policy **entirely from a fixed, pre-collected dataset** $\mathcal{D}$, with no further interaction with the environment. The dataset is generated by some **behavior policy** $\pi_\beta$, which may be suboptimal, heterogeneous, or even non-Markovian.

The standard objective is to find a policy $\pi^*$ maximising discounted cumulative reward:

$$\pi^* = \arg\max_\pi \; \mathbb{E}_\pi \left[ \sum_{t=0}^\infty \gamma^t r(s_t, a_t) \right]$$

within the MDP tuple $(\mathcal{S}, \mathcal{A}, p_0, p(s'|s,a), r(s,a), \gamma)$.

Standard approximate dynamic programming (e.g., Q-learning) requires evaluating the value of the **greedy action** at the next state:

$$\mathcal{L}_\text{TD}(\theta) = \mathbb{E}_{(s,a,s') \sim \mathcal{D}} \left[ \left( r(s,a) + \gamma \max_{a'} Q_{\hat\theta}(s', a') - Q_\theta(s,a) \right)^2 \right]$$

The $\max_{a'}$ operator will query $a'$ values that may be **out-of-distribution (OOD)** — i.e., actions not present in $\mathcal{D}$. This is dangerous because $Q_{\hat\theta}(s', a')$ is unreliable for OOD inputs, often producing **catastrophically overestimated** Q-values that drive learning off a cliff.

**The standard solutions** — constraining the policy to stay near $\pi_\beta$ (e.g., BCQ, BEAR, TD3+BC) or regularising Q-values to be low for OOD actions (e.g., CQL) — all impose an **improvement vs. safety trade-off**. The tighter the constraint, the safer but also the less the policy can improve.

### 1.3 IQL's Key Insight

> **Can we do multi-step dynamic programming entirely in-sample — without ever querying Q(s', a') for actions outside the dataset — while still learning a policy better than the behavior policy?**

IQL answers: **yes**, by reinterpreting the Bellman backup target.

Instead of:
$$V(s') \approx \max_{a'} Q(s', a') \quad \leftarrow \text{requires OOD queries}$$

IQL approximates:
$$V(s') \approx \max_{\substack{a' \in \mathcal{A} \\ \pi_\beta(a'|s') > 0}} Q(s', a') \quad \leftarrow \text{only in-support actions}$$

and achieves this via **expectile regression**, using only $(s, a)$ pairs that appear in $\mathcal{D}$.

## 2. Background: Expectile Regression

### 2.1 Definition

For a random variable $X$ and $\tau \in (0,1)$, the **$\tau$-expectile** $m_\tau$ is defined as:

$$m_\tau = \arg\min_{m} \; \mathbb{E}_{x \sim X}\left[ L_2^\tau(x - m) \right]$$

where the **asymmetric squared loss** is:

$$L_2^\tau(u) = |\tau - \mathbf{1}(u < 0)| \cdot u^2 = \begin{cases} \tau \cdot u^2 & \text{if } u \geq 0 \\ (1-\tau) \cdot u^2 & \text{if } u < 0 \end{cases}$$

**Intuition:**
- $\tau = 0.5$: standard mean squared error → recovers the **mean** of $X$
- $\tau \to 1$: heavily penalises underestimates → recovers the **supremum** of $X$
- $\tau \in (0.5, 1)$: interpolates between mean and maximum

**Key theorem** (Lemma 1 in the paper): For a bounded random variable $X$ with supremum $x^*$:

$$\lim_{\tau \to 1} m_\tau = x^*$$

This is the theoretical foundation of IQL. By choosing $\tau$ close to 1, the expectile approximates the maximum.

### 2.2 Conditional Expectile Regression

For pairs $(x, y)$ drawn from a joint distribution, we can fit **state-conditional expectiles**:

$$\arg\min_{m_\tau(x)} \; \mathbb{E}_{(x,y) \sim \mathcal{D}} \left[ L_2^\tau(y - m_\tau(x)) \right]$$

This is computable via stochastic gradient descent and is the building block for learning $V_\psi(s)$.

## 3. The IQL Algorithm: Full Derivation

IQL has **two stages**: (1) value learning via TD, and (2) policy extraction via advantage-weighted regression.

### 3.1 Stage 1 — Value Function Learning

IQL learns two functions simultaneously: a **state value function** $V_\psi(s)$ and a **Q-function** $Q_\theta(s,a)$.

#### 3.1.1 Learning V(s) via Expectile Regression

The value function is updated to estimate an **upper expectile** of the Q-values over actions in the dataset:

$$\boxed{ \mathcal{L}_V(\psi) = \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ L_2^\tau\!\left( Q_{\hat\theta}(s,a) - V_\psi(s) \right) \right] }$$

**Why this works:** For a fixed state $s$, the dataset contains many $(s, a)$ transitions with varying actions $a \sim \pi_\beta(\cdot|s)$. The Q-values $Q_{\hat\theta}(s,a)$ thus form a distribution over actions. Fitting $V_\psi(s)$ with the asymmetric loss $L_2^\tau$ makes it converge to the $\tau$-expectile of this distribution. As $\tau \to 1$, $V_\psi(s)$ approaches:

$$V_\psi(s) \longrightarrow \max_{\substack{a \\ \pi_\beta(a|s) > 0}} Q(s, a)$$

**Crucially**, this loss only uses actions $a$ that appear in $\mathcal{D}$ — no OOD queries.

**Note on a subtle issue:** One might instead write the expectile loss directly on the raw TD target $r + \gamma Q(s', a')$ sampled from $\mathcal{D}$. This would conflate two sources of randomness: (1) the action $a'$ and (2) the stochastic transition $s' \sim p(\cdot|s, a)$. A "lucky" next state could inflate the target even without a good action being available. IQL separates these by learning $V$ and $Q$ jointly (see below).

#### 3.1.2 Learning Q(s, a) via SARSA-style Backup

The Q-function uses a **standard MSE temporal difference loss**, but bootstraps off $V_\psi(s')$ rather than $\max_{a'} Q(s', a')$:

$$\boxed{ \mathcal{L}_Q(\theta) = \mathbb{E}_{(s,a,s') \sim \mathcal{D}} \left[ \left( r(s,a) + \gamma V_\psi(s') - Q_\theta(s,a) \right)^2 \right] }$$

**Why $V_\psi$ instead of $\max_{a'} Q$?** Because $V_\psi(s')$ is already a learned estimate of the best in-distribution action value, and it was computed without querying OOD actions. Using the MSE here also correctly **averages over the stochastic dynamics** $s' \sim p(\cdot|s,a)$, preventing the "lucky sample" inflation mentioned above.

This is analogous to a SARSA update, but instead of using the next dataset action $a'$, it uses the learned upper-expectile estimate $V_\psi(s')$.

**In practice:** IQL uses **clipped double Q-learning** — two Q-networks $Q_{\theta_1}, Q_{\theta_2}$ with the minimum taken when computing $V$ and $\pi$ targets, to prevent overestimation:

$$Q_{\hat\theta}(s,a) = \min\left(Q_{\hat\theta_1}(s,a),\; Q_{\hat\theta_2}(s,a)\right)$$

Target networks $\hat\theta$ are updated via **Polyak averaging**: $\hat\theta \leftarrow (1-\alpha)\hat\theta + \alpha\theta$.

### 3.2 Stage 2 — Policy Extraction via Advantage-Weighted Regression

Once $V_\psi$ and $Q_\theta$ have converged, the policy $\pi_\phi$ is extracted by **advantage-weighted behavioral cloning**:

$$\boxed{ \mathcal{L}_\pi(\phi) = -\mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ \exp\!\left(\beta \cdot \underbrace{\left(Q_{\hat\theta}(s,a) - V_\psi(s)\right)}_{\text{advantage } A(s,a)}\right) \log \pi_\phi(a | s) \right] }$$

where $\beta \in (0, \infty)$ is an **inverse temperature** hyperparameter.

**Derivation sketch:** This is the solution to the constrained optimisation:

$$\max_\pi \; \mathbb{E}_{(s,a) \sim \mathcal{D}}[A(s,a)]  \quad \text{subject to} \quad D_\text{KL}(\pi \| \pi_\beta) \leq \epsilon$$

The Lagrangian dual of this problem yields a policy of the form $\pi^*(a|s) \propto \pi_\beta(a|s) \cdot \exp(\beta \cdot A(s,a))$. Fitting a parametric $\pi_\phi$ to this target by minimising KL divergence gives exactly $\mathcal{L}_\pi(\phi)$.

**Intuition for the hyperparameter $\beta$:**
- $\beta \to 0$: ignores the advantage, reduces to pure behavioral cloning of $\pi_\beta$
- $\beta \to \infty$: concentrates mass on the single best action, approximates greedy policy improvement
- Practical values: $\beta = 3.0$ for locomotion, $\beta = 10.0$ for AntMaze

Note: this policy extraction step implicitly enforces a distributional constraint, but it does **not feed back** into the value function training — the value learning is fully decoupled.

## 4. Full Training Loop

```
Initialise: ψ (V-net), θ₁, θ₂ (Q-nets), θ̂₁, θ̂₂ (target Q-nets), φ (policy net)

─── TD Learning (IQL) ────────────────────────────────────────────────
For each gradient step:

  1. Sample minibatch B = {(s, a, s', r)} from D

  2. Update V-function (expectile regression):
       ψ ← ψ - λ_V · ∇_ψ L_V(ψ)
       where L_V = E_B[ L_2^τ( min(Q_θ̂₁, Q_θ̂₂)(s,a) - V_ψ(s) ) ]

  3. Update Q-functions (SARSA-style MSE):
       For i ∈ {1, 2}:
         θᵢ ← θᵢ - λ_Q · ∇_θᵢ L_Q(θᵢ)
         where L_Q = E_B[ ( r + γ·V_ψ(s') - Q_θᵢ(s,a) )² ]

  4. Soft-update target networks:
       θ̂ᵢ ← (1 - α)·θ̂ᵢ + α·θᵢ     [α = 0.005]

─── Policy Extraction (AWR) ──────────────────────────────────────────
For each gradient step:

  5. Update policy (advantage-weighted BC):
       φ ← φ - λ_π · ∇_φ L_π(φ)
       where L_π = -E_B[ clip(exp(β·A(s,a)), -∞, 100) · log π_φ(a|s) ]
       and  A(s,a) = min(Q_θ̂₁, Q_θ̂₂)(s,a) - V_ψ(s)

Note: Steps 3–5 can run concurrently for online fine-tuning.
```

## 5. Theoretical Justification

### 5.1 Convergence to the Optimal In-Distribution Value Function

**Theorem 3** (from the paper): Under the IQL update rules, the learned value function converges in the limit to the **in-distribution optimal value function**:

$$\lim_{\tau \to 1} V_\tau(s) = \max_{\substack{a \in \mathcal{A} \\ \pi_\beta(a|s) > 0}} Q^*(s, a)$$

where $Q^*$ is the optimal Q-function restricted to the dataset support.

**Proof sketch:**
1. **Lemma 1** guarantees that as $\tau \to 1$, expectile regression recovers the supremum of the target distribution.
2. **Lemma 2** establishes that $V_{\tau_1}(s) \leq V_{\tau_2}(s)$ whenever $\tau_1 < \tau_2$ (monotonicity of expectiles). This follows by a policy-improvement-style argument applied recursively to the Bellman equations.
3. **Corollary 2.1** shows $V_\tau(s) \leq \max_{a: \pi_\beta(a|s)>0} Q^*(s,a)$ for all $\tau$.
4. Combining monotone convergence with the upper bound yields Theorem 3.

### 5.2 Why Multi-Step DP Matters: The "Stitching" Property

**Single-step methods** (e.g., Decision Transformer, Onestep RL) fit $Q^{\pi_\beta}$ — the value of the behavior policy — and then take one step of policy improvement. This works when the dataset contains near-optimal trajectories, since $Q^{\pi_\beta}$ is already close to $Q^*$.

However, in tasks like **AntMaze**, the dataset contains no complete optimal trajectories — the agent must **stitch** together sub-trajectories that individually travel between different parts of the maze. There is no single trajectory from start to goal. In this regime:

- $Q^{\pi_\beta}(s, a) \ll Q^*(s, a)$ for most $(s, a)$ pairs
- Single-step methods see near-zero advantage everywhere → policy barely improves from $\pi_\beta$
- Multi-step DP methods **propagate reward signals backwards** through the Bellman operator, gradually building up accurate $Q^*$ estimates even when no single trajectory achieves the reward

IQL's SARSA-style backup with $V_\psi$ performs this multi-step propagation, which is why it dramatically outperforms single-step methods on AntMaze (+140 points in aggregate score, vs. +30 for the next best multi-step method).

## 6. Key Design Choices and Hyperparameters

| Design Choice | Value | Rationale |
|---|---|---|
| Expectile $\tau$ | 0.7 (loco), 0.9 (AntMaze) | Higher $\tau$ → closer to $\max$; needed more for stitching tasks |
| Inverse temp $\beta$ | 3.0 (loco), 10.0 (AntMaze) | Higher $\beta$ → more aggressive policy improvement |
| Target network update | $\alpha = 0.005$ (Polyak) | Stabilises TD learning |
| Q-function trick | Clipped double Q | Prevents overestimation |
| Advantage clipping | $\exp(\cdot) \leq 100$ | Prevents numerical explosion for high-advantage actions |
| Policy architecture | Gaussian, state-independent std | Simplicity; works well empirically |
| Optimiser | Adam, lr $= 3 \times 10^{-4}$ | Standard choice |
| Network size | 2-layer MLP, 256 hidden units | Consistent with D4RL baselines |
| Training budget | 1M gradient steps | Sufficient for convergence on all D4RL tasks |

**Critical note on $\tau$:** This is the most impactful hyperparameter. Fig. 3 of the paper shows that on antmaze tasks, $\tau = 0.5$ (i.e., pure SARSA, which recovers $V^{\pi_\beta}$) achieves near-zero performance, while $\tau = 0.9$ achieves 70–100. The stitching capability is entirely absent without a sufficiently large $\tau$.

## 7. Connection to Other Methods

| Method | Relation to IQL |
|---|---|
| **SARSA** | IQL with $\tau = 0.5$ |
| **Q-Learning** | IQL with $\tau \to 1$ |
| **BCQ** | Similar constraint philosophy (in-support $\arg\max$), but uses a generative model; can still produce OOD samples |
| **CQL** | Penalises Q-values of OOD actions explicitly; more expensive (requires extra loop over all actions) |
| **BEAR** | Constrains policy to MMD ball around $\pi_\beta$; requires density estimation |
| **AWR / AWAC** | IQL shares the policy extraction step; IQL improves the value function training |
| **Onestep RL / DT** | Single-step methods; no multi-step DP; fail on stitching tasks |

The standard offline RL benchmark used to evaluate IQL. Key dataset families:

| Domain | Key Challenge | IQL Performance |

| **Gym-MuJoCo** (hopper, walker2d, halfcheetah) | Suboptimal data; narrow distributions | Competitive with CQL (~692 total) |

**Normalisation:** D4RL reports scores on a 0–100 scale where 0 = random policy and 100 = domain expert. This allows comparison across tasks.

## 9. Summary

IQL makes one elegant observation: the Bellman backup's $\max_{a'} Q(s', a')$ can be replaced by an **expectile of Q over in-distribution actions**, estimated via an asymmetric squared loss. This single change eliminates all OOD action queries from value function training, removes the need for explicit constraints or density models, yet still enables multi-step dynamic programming.

The result is an algorithm that is:
- **Simple**: minor modification over standard SARSA-style TD
- **Fast**: ~20 min for 1M updates on a single GPU (4× faster than CQL)
- **Effective**: state-of-the-art on D4RL, especially on stitching-heavy tasks
- **Fine-tunable**: strong initialisation + AWR policy extraction = effective online fine-tuning
